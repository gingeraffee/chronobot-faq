What is ChronoBot? Getting Started

ChronoBot helps servers track events with countdowns and reminders. It keeps one pinned “source of truth” for upcoming dates, plus automated reminders so your server doesn’t get haunted by “wait when is it again?” every five minutes.

What do I need to set up first? Getting Started

Three-step setup:

  • Pick your events channel → /seteventchannel
  • Add an event → /addevent
  • Confirm the pinned countdown appears and updates

If the pinned message doesn’t appear, it’s almost always a channel permission issue. Check View Channel, Send Messages, Embed Links, Read Message History, and Pin Messages/Manage Messages.

Command cheat sheet (most used) Commands

Setup: /seteventchannel • /healthcheck

Create: /addevent

Browse: /listevents • /nextevent • /eventinfo

Edit: /editevent • /dupeevent • /removeevent

Reminders: /remindall • /setmilestones • /resetmilestones • /silence

Customization: /theme • /banner set • /banner clear

Supporter: /vote • /vote_debug

Maintenance: /update_countdown • /resendsetup • /purgeevents

What are “Supporter Perks” (vote-locked features)? Supporter

ChronoBot is free to use, and the core countdown system is available to everyone. “Supporter Perks” are optional upgrades that unlock after you vote for ChronoBot on Top.gg.

Run /vote for the vote button and to confirm your supporter status.

How do I unlock vote-locked features? Supporter

1) Run /vote in your server.
2) Tap the Vote button (Top.gg will open).
3) Return and run /vote again to confirm you’re unlocked.

If Top.gg needs a moment to update, wait a minute and retry.

Which features are vote-locked (Supporter Perks)? Supporter
  • Themes: /theme
  • Advanced milestones: /milestones advanced
  • Templates: /template save + /template load
  • Banners: /banner set
  • Weekly digest: /digest enable

These are quality-of-life upgrades: less repetition, more customization, and nicer presentation.

I changed the theme, but the pinned embed didn’t change. Why? Troubleshooting

Theme changes affect how the pinned countdown embed is rendered. If it doesn’t update immediately, try /update_countdown or wait for the next auto-refresh cycle.

If it still doesn’t change: run /healthcheck and confirm Chromie can Read Message History and Embed Links in the event channel.

How do banners work (and what kind of URL do I need)? Customization

Banners add a hero image to the pinned countdown embed. ChronoBot usually displays the banner for the next upcoming event that has a banner set.

Use a direct image URL ending in .png, .jpg, .jpeg, or .webp.

Example: /banner set index: 1 url: https://example.com/mybanner.png

If your banner isn’t showing:

  • The link isn’t a direct image file
  • The banner is on an event that isn’t the next upcoming event
  • The event is in the past
  • The bot lacks Embed Links permission
Which permissions does ChronoBot need? Permissions

In the countdown channel, ChronoBot typically needs:

  • View Channel
  • Send Messages
  • Embed Links
  • Read Message History (to find/update its pinned message)
  • Pin Messages or Manage Messages (if pinning is used)
  • Manage Messages (to auto-delete milestone announcements and repeating reminders after the event)

If commands work but nothing posts, check channel permission overrides first. Discord’s permission layering is… dramatic.

“I couldn’t create or access the pinned countdown message.” What does that mean? Troubleshooting

That message almost always means a permissions issue in the countdown channel. ChronoBot can’t create the pinned message, can’t read history to find it, or can’t pin/update it.

Fix: confirm Send Messages, Read Message History, Embed Links, and Pin Messages/Manage Messages.

How do events work in ChronoBot? Events

ChronoBot stores events (name + date/time + optional settings) and shows them in a numbered list. Many management commands reference events by index to keep things fast.

Best practice: keep names specific (“Guild Raid Night”, “Giveaway Ends”, “Release Day”) so reminders stay readable.

What happens when an event passes? Events

When an event passes, the pinned countdown will move on to the next upcoming event. To keep the channel tidy, ChronoBot can also clean up reminder posts after the event.

Reminder cleanup: milestone announcements and repeating reminders stay visible while the event is upcoming, then they’re deleted 24 hours after the event time (if the bot has permission to delete messages).

How do reminders work? Reminders

Reminders can fire automatically at milestone intervals (days before the event), or manually with /remindall. You can customize milestone days per event and disable reminders for an event without removing it.

If reminders don’t ping who you expect, confirm role mentions are allowed and the bot is permitted to mention them in that channel.

Do milestone announcements and repeating reminders get deleted? Reminders

Yes. To prevent the countdown channel from getting cluttered, ChronoBot can automatically clean up its reminder posts.

How it works:

  • Milestone announcements and repeating reminders stay visible while the event is upcoming.
  • After the event time passes, ChronoBot deletes those reminder posts 24 hours later.

Required permission: ChronoBot needs Manage Messages in the countdown channel to delete messages. If the bot can’t delete messages due to permissions, the reminders will remain.

Note: Auto-cleanup applies to reminder messages ChronoBot can track. Older reminder messages posted before this feature may require manual cleanup.

What are repeating reminders? Reminders

Repeating reminders post on a schedule (daily/weekly/etc.) for an event, in addition to milestone reminders. Enable with /setrepeat and disable with /clearrepeat.

Why don’t I see autocomplete for event index fields? Troubleshooting

Autocomplete only appears if the command option is wired to an autocomplete handler. If you expected autocomplete and it’s missing, it may be a command-definition issue or a Discord client quirk.

Report it in the support server with the command name, screenshots, and whether you’re on desktop/mobile.

Best practices (recommended server setup) Suggestions
  • Create a dedicated events channel (ex: #events or #countdowns).
  • Keep event names specific so reminders make sense.
  • If you use role pings, create one role (ex: “Event Pings”) and let people opt in.
  • Use /dupeevent for recurring formats so you don’t rebuild everything.
  • Use a banner only for “headline” events so the pinned embed stays clean.
What data does ChronoBot store? Privacy

ChronoBot stores server configuration needed to function (like which channel to post in) and event/reminder data you create. It does not read your private messages for content.
